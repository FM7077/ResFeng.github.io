一些关于做微信小程序时遇到的问题及一些小技巧

顺便再次吐槽一下, 官方文档写得太烂了

Toast 现已更改为 showToast

js 中动态更改组件属性

  1. 组件要动态变化的属性设置一个别名
  2. 在 js 文件中的 data 里新建一个变量, 与别名一样的名字
  3. 在函数中动态更改

监听器里调用上一级变量

在监听器或定时器里使用的参数变量定义时必须加 const 关键字

this 转换为 that, 用来访问上一级的变量

输入框剔除无效字符

点击按钮后获取输入框内容

一种是把输入框和按钮放在一个表格即

中, 然后输入框设置别名, 表格绑定事件, 在事件回调函数中通过别名获取输入框内容

另一种是每个输入框都绑定事件, 同时为每个输入框设一个变量来实时接收输入框的文字, 再在按钮的事件中通过变量来间接读取

模板引用

  1. 先建立模板: xxx.wxml
  2. 在要使用这个模板的页面引用模板: <import src=”../../xxx.wxml> 并在该页面使用模板

Please do not register multiple Pages in undefined.js 错误

检查 app.js 中是否有 Page({}) 删去即可

关于 input 的 maxlength

在模拟器上测试是正常的, 但在手机上测试时, 得输入到 maxlength 指定的长度才不会被认为是空的.

为解决此 bug, 直接不采用 maxlength, 而是在监听器里实时判断输入字符长度, 截取规定长度字符串

flex 流布局的居中要点

水平居中: justify-content: center;

垂直居中: align-items: center;

控件要设置宽和高 !

中止代码执行

需求: 有时候需要判断是否能联网, 用户是否存在, 若为否则跳转到404界面或其他错误界面, 并同时中止小程序继续进行

例: 在微信小程序加载的时候判断是否能连上服务器, 若不能则跳转 error 界面

如果不进行特别处理, 其余代码还是会被运行

emmmmm, 傻逼了, 其实只要把其余代码放在 if 里就可以了

然后这个时候你会发现 else 中的跳转并不起作用

然后我就用了 ==wx.reLaunch== 就可以跳转了, 原因暂不知. 关于 wx.reLaunch 的介绍见下面

你可能不知道的多种跳转方法

官方文档

wx.navigateTo, wx.redirectTo, wx.reLaunch

一般用的可能 wx.navigate 比较多, 它可以跳转到5层级以内的任何页面, 这种跳转方法不会销毁其他页面, 跳转后也能返回上一层页面, 超过5个层级不会跳转.

为了解决这个问题, 找到了 wx.redirectTo, 它可以跳转到任何层级的页面, 该跳转方法会销毁上一层的页面( 不是所有 ), 因此跳转过去后无法返回到上一层页面. 对于不想再返回的页面, 导向的时候就可以用 wx.redirectTo.

wx.reLaunch: 清空页面栈, 导航到指定页面.

遇到 bug

本地测试 localhost / 服务器本地ip

用模拟器测试时如果请求服务器链接为 localhost 那么没什么问题, 但如果用的是服务器本地 ip 时会报503 错误, 但服务器又能收到模拟器发出的消息, 真是神奇.

如果用真机测试那么当然不能写 localhost 了, 而应该写 服务器本地 ip, 这时就很正常~ 就是很神奇
